# 📄 Desafio 018 - Módulo 05 - Capítulo 29

## 📌 Enunciado

Desafio proposto ao final do capítulo **"Projeto Portfólio"** do curso de HTML e CSS da plataforma **Curso em Vídeo**.

## 💡 Objetivo

Criar uma página de **portfólio pessoal**, reunindo informações de apresentação, vídeo, skills, formação acadêmica e projetos desenvolvidos, utilizando conceitos de **HTML5**, **CSS3**, **Grid Layout**, **Flexbox**, responsividade, tema claro/escuro e interações com JavaScript.

## 🌐 Visualização

👉 [Projeto Portfólio](https://giulia-mb.github.io/html-semantico/modulo-05-html5-css3/desafios/d018/index.html)

## 🛠️ O que foi praticado

* Estruturação da página com tags semânticas
* Uso de `header`, `nav`, `main`, `section`, `article` e `footer`
* Criação de layout responsivo com **CSS Grid**
* Organização de elementos internos com **Flexbox**
* Menu fixo no topo com `position: sticky`
* Navegação por âncoras internas
* Uso de ícones com **Font Awesome**
* Criação de ficha pessoal com imagem, texto e links sociais
* Incorporação de vídeo do YouTube com `iframe`
* Criação de barras de progresso para skills
* Organização da formação acadêmica em cards
* Listagem de projetos com imagem, descrição e links externos
* Uso da tag `details` e `summary` para expandir informações
* Criação de tema claro e escuro com variáveis CSS
* Uso de JavaScript para alternância de tema
* Uso de `localStorage` para salvar a preferência do tema
* Aplicação de transições, hover, bordas e cores personalizadas

## 📚 Aprendizados

* Um portfólio pode reunir diferentes seções em uma única página de forma organizada
* CSS Grid facilita a criação de layouts com colunas principais
* Flexbox ajuda a alinhar e distribuir elementos dentro dos cards
* `position: sticky` permite manter elementos visíveis durante a rolagem
* Variáveis CSS tornam a troca de tema mais simples e organizada
* JavaScript pode ser usado para alterar classes e modificar o visual da página
* `localStorage` permite salvar preferências do usuário no navegador
* A tag `details` ajuda a mostrar mais informações sem ocupar muito espaço inicialmente
* Ícones tornam a navegação mais visual e compacta
* Cards deixam informações de formação e projetos mais fáceis de visualizar
* Responsividade é essencial para que o portfólio funcione bem em diferentes telas

## 📄 Estrutura do desafio

### `index.html`

Página principal contendo:

* menu de navegação com ícones
* ficha de apresentação pessoal
* links para redes sociais
* seção com vídeo de apresentação
* seção de skills com barras de progresso
* seção de formação acadêmica
* seção de projetos com imagens, descrições e links
* rodapé com créditos

### `style.css`

Arquivo responsável pela aparência e organização visual:

* reset básico dos elementos
* criação de variáveis CSS em `:root`
* configuração de tema claro e escuro
* layout principal com CSS Grid
* organização dos cards com Flexbox
* menu fixo com `position: sticky`
* estilização da ficha pessoal
* estilização do vídeo responsivo
* barras de progresso das skills
* cards de formação e projetos
* responsividade com `@media`
* efeitos de hover e transições

### `script.js`

Arquivo responsável pelas interações:

* alternância entre tema claro e escuro
* troca do ícone do botão de tema
* salvamento da preferência no `localStorage`
* rolagem suave para links internos do menu

## 🚀 Resultado

O desafio resultou em uma página de portfólio pessoal responsiva, com apresentação profissional, seções bem organizadas, projetos clicáveis, tema claro/escuro e interações simples com JavaScript.
